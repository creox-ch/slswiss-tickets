# Предкоммитная проверка (slswiss-tickets)

Порядок из [`CLAUDE.md`](CLAUDE.md), с граблями этой машины.

## 1. Синтаксис серверных файлов

```powershell
node --check app/api/<...>/route.js
node --check lib/<...>.js
```

Проверяй те файлы, которые трогал, — не весь репо.

**Локального `node` на этой машине может не быть** (в Cowork его нет). Тогда
гейт — GitHub Actions: синтаксическая ошибка валит все тесты. Не подменяй
проверку словами: если `node --check` не запускался, так и скажи и дождись CI.

## 2. Тесты

```powershell
npm test                                   # весь набор
npx playwright test tests/unit/forms.spec.js   # точечно
```

E2E мокают API — внешние сервисы и ключи не нужны.

Без node локально статус CI читается через браузер: Actions → нужный run на
github.com. `gh` CLI на машине нет, API без токена логи не отдаёт и упирается
в лимит 60 запросов/час. Идти и смотреть самому, а не ждать, что отчёт
принесут.

## 3. Показать диффы

```powershell
git -C "<путь к репо>" status --short
git -C "<путь к репо>" diff --stat
```

Покажи это пользователю **до** коммита. При неочевидных правках — и полный
`git diff` по файлу.

## 4. Коммит

Git — **только из PowerShell**, не из bash-песочницы: mount не даёт unlink и
оставляет залипший `.git/index.lock`. Если лок остался с прошлого раза:

```powershell
Remove-Item "<путь к репо>\.git\index.lock" -Force
```

Один коммит на одну осмысленную правку. Сообщение по-русски, в стиле репо:
`feat(forms): …`, `fix(forms): …`, `chore(skills): …`. Многострочные сообщения
передавай через single-quoted here-string (`@'` … `'@`, закрывающий `'@` — в
нулевой колонке).

Заканчивай сообщение строкой:

```
Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

## 5. Ветка или main

⚠ На `main` в `creox-ch/slswiss-tickets` включено правило GitHub
**«Changes must be made through a pull request»**. Прямой пуш формально
проходит — у токена есть право bypass — но каждый раз это обход защиты, и
GitHub пишет в ответ `Bypassed rule violations`.

**По умолчанию: ветка + PR.**

```powershell
git -C $r checkout -b <тип>/<короткое-имя>
git -C $r push -u origin <ветка>
gh pr create --fill     # gh на машине может не быть — тогда ссылку даёт push
```

Прямо в `main` пушить только если Иванна на этот раз явно сказала. Не считай
прошлое разрешение бессрочным — правило защиты появилось не просто так.

Пуш в `main` = авто-деплой на Vercel (https://slswiss-tickets.vercel.app).
Для правок, затрагивающих приём заявок или оплату, скажи об этом вслух до пуша.

## Чего не делать

- Не коммитить `.env*`, ключи, токены, тестовые дампы базы.
- Не добавлять `-A` вслепую — смотри `git status` и добавляй нужное.
- Не использовать `--no-verify` и не отключать подпись.
- Секреты в Vercel/Supabase/Payrexx вводит пользователь сама, не агент.
